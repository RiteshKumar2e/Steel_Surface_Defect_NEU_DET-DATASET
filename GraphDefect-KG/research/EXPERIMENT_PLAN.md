# Experiment Plan — GraphDefect-KG

## 1. Data protocol
- **Dataset:** NEU-DET, 6 classes × 300 images = 1,800.
- **Splits:** stratified 70 / 15 / 15 (train / val / test). Report **5-fold** stratified CV
  and **3 random seeds** for final numbers; the notebook uses a single split for speed.
- **Preprocessing:** fixed per `METHODOLOGY.md`; identical across all models.

## 2. Models to evaluate
| Group | Models |
|-------|--------|
| Classical ML (on MobileNetV2 global features) | KNN, SVM, Random Forest, XGBoost* |
| CNN | Base CNN, MobileNetV2, EfficientNet-B0*, ResNet50* |
| Graph | GCN, GraphSAGE, GAT, GIN* |
| Hybrid | MobileNetV2+KNN-graph, MobileNetV2+GCN, MobileNetV2+GAT, +handcrafted+GCN, +GNN+KG, **proposed full hybrid** |

`*` = optional / resource-dependent.

## 3. Metrics
Accuracy, precision, recall, F1 (macro & weighted), ROC-AUC (OvR), PR-AUC, confusion matrix.
Efficiency: parameter count, model size, inference time, graph-construction time, graph
inference time, peak memory. Localisation (where annotations used): IoU, mAP, localisation
accuracy.

## 4. Training configuration (defaults)
- Optimizer: Adam, lr 1e-3, weight decay 5e-4.
- Epochs: ≥ 50 for reported runs (notebook default 25 for smoke tests).
- Batch: 8 graphs (graph-level), early stopping on val macro-F1.
- Backbone: frozen MobileNetV2 for the main comparison; a fine-tuned variant as an ablation.
- Hardware: CPU-compatible; GPU used automatically if available (`DEVICE=auto`).

## 5. Procedure
1. Extract features once, cache per image (embedding + handcrafted + graph).
2. Fit classical baselines on cached global features.
3. Train graph/hybrid models with the manual graph collate (notebook §16–17).
4. Evaluate on the held-out test set; log all metrics to a results table.
5. Repeat across seeds/folds; report mean ± std.
6. Run the ablation matrix (`ABLATION_STUDY.md`).
7. Run explainability faithfulness checks (deletion/perturbation).

## 6. Statistical reporting
Report mean ± std across seeds. Use paired tests (e.g. Wilcoxon signed-rank across folds)
before claiming one model beats another. State the significance threshold in advance.

## 7. Reproducibility checklist
- [ ] Fixed seeds recorded.
- [ ] Split indices saved.
- [ ] Feature cache versioned.
- [ ] Every table regenerated by executing the notebook (no manual numbers).
- [ ] Environment pinned (`backend/requirements.txt`).
